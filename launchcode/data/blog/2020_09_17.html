A Full Stack

September 17, 2020

Has it been three weeks?!

Well, I have been hard at work. Last week I found out that switching to PostgreSQL would be a wise decision for many reasons including a software package called PostGIS which adds spatial database extensions to Postgre, Node Postgres which offers a thorough collection of Node modules for interfacing with Postgres, and Sequelize which provides an ORM similar to how Hibernate is used in Java. Combine this with Angular (which uses Express) and run it on a Raspberry Pi 4 (because ARM architecture is the future) using Manjaro Linux (Arch Linux distros, for the win!) and we have a full stack on an ideal platform that show the potential for a lot of technologies.

That's a lot of stuff to describe for a project that has limited time left, and I haven't even mentioned the front-end map layers

There's also a few security features like 12-factor authentication, but we'll talk about that some other time.

This is a full-stack application! I should do a better job of writing down all the steps to make something like this, including how to use HTTPS instead of HTTP (because, I'm sick of tutorials that make a big deal about being about to build an application that runs on localhost, when a better solution is to run an application that is on the same network as your local host (your computer). It's why I put in the extra effort, and complain about why Google not letting you run your HTTPS server with your own signed certificates is a bunch of nonsense! (Of course my local certificates are going to be "insecure" by Google's standards. IT'S RUNNING ON MY LOCAL NETWORK!)

Knowing how to handle network issues is the next level up from knowing how to handle computer issues, because the router is basically the computer that allows every other computer in the house to communicate with the outside world. If you've had service done with a company like AT&T or Spectrum or Verizon, you know that the only thing they really do these days is swap out the box with a different box and transfer settings. The old box gets junked, which seems like a waste of resources.

That's kind of why I want to get back into learning computer networking, but not with Cisco this time but something that actually has practical applications: GNS3.

I'm not sure if I mentioned this before on this blog, but I've never really seen too many local companies in the area with hardware that has the Cisco logo on it. So a better investment would be to learn how to work with networking software that works on many devices rather than one specific brand.

It's a lonely path. It might not have any big brand names or logos tied to it, but much like everything else I work with, I can find out how to fix it easily and it just works!

A quick aside: last week, I rebuilt one of my Java assignments in Typescript. Not only was this helpful in my understanding of TypeScript, but it shows that Java can be replaced with TypeScript. Well, except for the hashcode functions. Which I guess I'll figure that out some other time.
