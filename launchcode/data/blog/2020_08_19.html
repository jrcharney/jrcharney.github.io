Stewie

August 19, 2020

There are two projects that I like that allow for using Spotify on the command line in Linux. They both are written in Rust, they both use Cargo as a package manager, they both are available crates.io, and they both need to be forked.

Enter a new project I a proposing called Stewie. Stewie (which is a better name than STUIE (Spotify Textual User Interface Enhanced) is an amalgamation of Rigellute's spotify-tui and hrkfdn's ncspot. Both of these applications are 100% written in Rust, both of them use the cursive library (which is based on ncurses), both of them have flaws.

spotify-tui

Pros

spotify-tui is laid out in the familiar desktop interface format that the official Spotify desktop application uses.

Cons

spotify-tui uses an external project called rspotify for authentication. rspoitfy requires the user to set up a developer account to get API credentials. The user shouldn't have to do this.

ncspot

Pros

ncspot does not require you to set up a Spotify developer account. You log in with your typical user credentials.

The layout is in tabs, which I think is a good idea if you are running this app in frame in a terminal multiplexer or are using a small screen, it needs improvement.

Cons

In some terminal setups, ncspot will not work. You will get a message saying it is incompatible. This happens if you modify your terminal to do things like use tmux or use a terminal that supports 256 colors or more.

The keybindings were also cantankerous. This is what made using ncspot difficult.

stewie

Both of these applications have their strenghts and weaknesses, and their developers seem to have different ideas as to how their applications can work. So what can we do to fix this problem?

stewie aims to be the best of both applications. The great thing about github is that you can fork other people's projects and make a better application of your own.

I wish this could have been my LaunchCode capstone project, but it's a little late in the game to pivot, on top of that the LaunchCode folks seem interested in developers make business applications. Requirement: Spotify Premium

As with any Spoitfy application written by a third party, this application will only work if you have a premium account, which honestly for $10 per month is more than a fair price for access to all the streaming music and podcasts you can have access to.

And yes, there's the argument that the musicians are still not getting their fair share of the revenue. Blame the record labels for that not the people who make access to the music available through streaming. This was the same problem 20 years ago where the record labels had been screwing the artists when along comes Napster to be the perfect scapegoat.

If the same problem from 20 years ago exists that was also around 30 years ago and prior, maybe it isn't the fault of the technology that distributes the artist music but the fault of the record label who takes in all the revenue from Spotify but still give their artists a fraction of a cent each time a song is played to begin with.

The best thing you can do is support independent artists, go to their concerts, and buy a T-shirt or physical record.

Streaming music platforms are a convenience compared to the old days of tap players, CD players, piracy, and radio.

Spotify is not the bad guy here. The recording industry has, is, and will always be the villain so as long as the industry doesn't pay artists their fair share.

Features

I've stated some of the things I want to with this app already, but let's make sure I didn't skip over anything so I remember to add them to my application in the future.

  • Responsive Application Design (RAD) - Much like web applications have Responsive Web Design (RWD), stewie will make use the layouts used in spoitfy-tui and ncspot. One for large window space the other for small window space.
  • User-friendly Login - ncspot's greatest strength. No need to set up a developer account. Save those developer slots for your own applications. Maybe even a fork of stewie.
  • Familiar Keybindings - There's some software that lets you use media keys to control playback. Hope to integrate that.
  • Easy to Use! - Use Tab and Shift+Tab to move between things.
  • Basic visualization - They'll likely work best on a computer running the application, not so much if you remote into it. All the more reason for me to test this on my hacktop.

The Future

stewie will eventually have a GUI version similar to Webamp (or Desktop Webamp) that will have all the Milkdrop/Butterchurn visualization so that a new generation of music fans, artists, and demoscene enthusiasts can enhance their musical experience.

While Webamp is a Node.js application, in the interest of using Rust I'm going to see if I can use a Rust GUI library.. There are many options. I need to decide which would be best. At any rate stewie-gui (which could be called brian) will be a separate project and one that I hope will not take as long as that novel Brian has been working on.

A Trello Board has been set up, but there probably won't be any major developments until after I finish LaunchCode Liftoff.

There also might be some tmux status bar integration with zscroll. I might rewrite that in Rust.

Keep an eye on this github repository for progress on this project.
