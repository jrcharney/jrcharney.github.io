So what is the "Pivot" part of this project? As I had mentioned, I want to shift from Angular to Express. Express is used by most of the popular frameworks (Angular, React, Vue), and just about every example I come across has stuff that uses Express.

The way Angular explains routing (that LaunchCode faltered with in that department) is that everything runs on a single-page application (SPA). This is problematic as your hands are kind of tied to one location unless you can set up the routing, and the other reason Angular doesn't explain it very well.

On the other hand, Express has a straightforward explanation which aligns with what I learned with Java Spring Boot. You set what kind of request you want (mostly GET and POST which correspond with get() or post()), you tell it the path ("route") where it should happen and what function to run when it goes there.

Similarly, Sequelize seems to apply itself similar to how Hibernate is used in Java with Spring Boot, linking the database with the application. (Spring Model. Sequelize Model.) I'm certain there is some form building. It just feel like a lot of the same stuff I picked up while learning Java could have been applied in another class for Node and JavaScript. It would probably be an advanced course. An LC201 class, if it ever existed.
