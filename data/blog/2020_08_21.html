Speaking of Angular, I realized that all the projects here are HTTPS, however if you work on a project via localhost, you can't use some of the features that are exclusive to HTTPS that are available on some browsers.

To rectify this problem, I will need to do a couple things.

First, I could probably set up Node in general to support HTTPS. Richard Russell wrote an article on how to do this, but it has two flaws:

  1. It's meant for developing Angular on Windows. Which is a terrible idea.
  2. It doesn't mention anything about the second solution that I will mention in a bit.

The good part about Richard's explaination is that it does explain hwo to create an HTTPS certificate. If you own your own web domain, your hosting provider might offer using a service like Let's Encrypt, which you should use.

The second solution is to use https-localhost, which does a lot of what Richard talks about.

It's probably a good idea to install https-localhost from NPM, as it might allow you to use the SSL features in Angular that allow you to run ng serve with your HTTPS certificate.

At any rate, I'll probably add more to this article later since something about my Manjaro Linux setup won't let me access my Angular program from another computer. It's probably another firewall issue set up by PAM. It's good that there is security on this thing, but there needs to be some easy instructions to allow for exceptions. I don't seem to have this problem with SSH or Samba.
