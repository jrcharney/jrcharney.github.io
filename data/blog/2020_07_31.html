Liftoff!

July 31, 2020

This website continues to grow. The process of making it look more pleasing is one that does take time to construct.

Slowly but surely this site becomes better than it was before.

Starting August 3, I will take part in an eight nine week course called Liftoff. A page is being set up to track the progress.

Efforts will also be made to move the grunt work of this site from my laptop to my hacktop. Yes, the Hacktop page now exists! I've had one of those embosed labels made with a label maker that was on the back side of my hacktop for years. But thanks to some motivation by LaunchCode, it exists. I'll write more about my hacktop on that page.

Python makes turing any computer into a makeshift server super easy.

It seems like such a downgrade, but actually, using editors like Vim in a terminal multiplexer (tmux) over Secure Shell (SSH) to work on a Git repository is pretty much the same thing. Alternatively, I could figure out how to use my hacktop's Raspberry Pi (or another Pi) as a file server with Samba..

There's a ton of stuff that I could do. I'd feel more confident about completing them if I had more resources to do them. Otherwise, it feels overwhelming sometimes.

One of my goals with my Liftoff project is that I would like to work with a team. I understand that doing so can be complex and difficult at times. However, given that these are not normal times, and everyone is pretty much separated by distance thanks to the ongoing pandemic, it is necessary for team members to work in technological environments and work on their communication skills with their team members over the Internet.

I would hope that working on my one of my Raspberry Pis this weekend will provide some idea and perspective on how to do this, more so since we will need to use Git to build this project. On the other hand, a private server which would provide the services we need to run some of the frameworks to get things done would also be of benefit.

I'm starting to regret non completing my Cisco CCNA certification last year. At the same time, this would be an oppotunity to review some of that and think of how one would establish a server for things some webhosts won't provide unless their is a significant markup.

Using the Pi might allow for a blueprint for the future.

At any rate, look out for some new content on this site.
